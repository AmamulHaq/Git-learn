git config --global user.name "Your name"
git config --global user.email "Your name"

Remote: The projects or repositories which are on github account are called remote repo.
Local: The projects which are on your computer system yet to be commited and pushed are known as local projects.
Clone: When we want to fetch the repository files we use clone command to clone remote repo into local space.
Clone command:
git clone <--url-->

To check status about

-which files are untracted
-which files are to be commited
-which files are modified
We use this command:
git status

when you are done with the coding you can use these commands and steps to upload your work in github.
Step 1: Stage your file
Staging is nothing but to save your file in local device as a file ready to commit. Think of it like dragging your few out of many projects in code editor to the stage so that you can publish those chosen one in open source.

Use this command to stage your file

git add <-file_name->

Step 2: Commit your file
When you commit your file it is almost into your remote repository. When we commit any file it means we are done with modifying content in our project. We take the real advantages of commit when we create different branches in repositories to use unique hash of commited file. Each commited file has a unique hash with it.

Use this command to commit your files

git commit -m <-message for what changes you made this time in this project->

Step 3:Push committed file
This is final step when you are working in any project. After you push your file it gets uploaded in your github account with amazing looking green box in your github profile.

Use this command to push your commited file

git push origin main

To unstage the staged files we use

git restore --stage <-file_name->

To stage all files we use

git add .
